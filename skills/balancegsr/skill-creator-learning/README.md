# skill-creator-learning

根据你的学习需求，生成一个定制化的学习项目 Skill。

生成的 Skill 会在你的整个学习过程中持续引导——从制定学习计划、逐个学习点推进、模块复盘，到最终的知识沉淀。不是一次性的学习建议，而是一个贯穿项目全程的结构化学习助手。

## 快速开始

安装后，对 agent 说：

- "我想系统学分布式系统"
- "帮我生成一个学习 skill"
- "创建学习项目"
- "I want to learn Rust systematically"

Creator 会用 2-3 轮对话了解你的情况（学什么、目标、基础、时间），然后生成一个定制的 SKILL.md。

## 它会问你什么

| 信息 | 为什么需要 | 示例 |
|---|---|---|
| 学习主题 | 确定领域和方向 | "分布式系统"、"RAG pipeline" |
| 学习目标 | 决定深度和验收标准 | "能独立设计中等复杂度的分布式方案" |
| 现有基础 | 确定起点，避免重复 | "有后端经验，用过 Redis 但没学过理论" |
| 时间范围 | 决定项目结构和粒度 | "3 个月，每周 8-10 小时" |

不会问太多。验收方式偏好、学习风格之类的，如果你在对话中自然提到了会采用，没提到就用合理默认值。

## 两种模式

| | 轻量模式 | 完整模式 |
|---|---|---|
| 适合 | ≤2 周的短期集中项目 | 1 个月以上的系统学习 |
| 模块数 | 2-4 个 | 4-8 个 |
| 复盘 | 按需（3+ 学习点的模块才建议） | 每个模块都建议复盘 |
| 结项 | 简要项目总结 | 完整认知框架文档 |
| 文件管理 | 全量加载（文件少） | 三层优先级动态调整 |

Creator 会根据时间范围推荐模式，你可以覆盖。两种模式在覆盖范围内的引导质量一致——轻量不是"缩水版"，是为短期项目裁剪了后期才需要的机制。

## 生成物长什么样

生成的项目 Skill 包含以下结构：

```
learn_[主题]/
├── .skill/
│   ├── SKILL.md              # 主控逻辑：启动协议 + 四模式引导
│   └── references/
│       ├── review_guide.md    # 复盘执行指南
│       └── framework_guide.md # 认知框架生成指南（仅完整模式）
```

安装后，Skill 自动引导你的学习项目走完四个阶段：

1. **规划模式** — 和你一起定义目标、拆解模块、生成学习计划
2. **学习模式** — 逐个学习点执行五步闭环（讲解→讨论→确认→验收→笔记）
3. **复盘模式** — 模块完成后整合知识、识别缺口、调整策略
4. **结项模式** — 生成认知框架（完整模式）或项目总结（轻量模式）

项目推进中，Skill 自动管理文件：保存笔记、追踪进度、调整读取优先级。你不需要手动操作任何文件。

## 示例：生成物的项目信息区

以"系统学习 RAG"为例，轻量模式生成的 SKILL.md 开头：

```yaml
---
name: learn_rag
description: >
  RAG（检索增强生成）学习项目引导。当用户在该项目目录中开始新对话、
  继续学习、进行复盘或结项时触发。
tools: [write, bash, web_search]
---
```

```markdown
## 项目信息

- 学习主题：RAG（检索增强生成）
- 学习目标：能用 LangChain 或 LlamaIndex 搭建可用的 RAG pipeline
- 现有基础：熟悉 Python，懂 LLM 基本原理
- 时间范围：1 周集中突破
- 项目模式：轻量模式
```

## 交付方式

生成完成后你可以选择：

- **直接创建** — 在当前目录的上级创建项目文件夹，Skill 直接可用
- **ZIP 打包** — 生成 zip 文件，可自行解压到任意位置或分享

## 兼容性

主要为 OpenClaw 设计，同时兼容能操作本地文件系统的 AI Agent 产品（Claude Code、WorkBuddy、CodeBuddy 等）。

- 所有指令使用描述性语言，不依赖特定平台 API
- 文件路径使用相对路径
- 文件格式统一为 Markdown
- frontmatter 为标准 YAML，非 OpenClaw 环境下 agent 忽略 frontmatter 仍可读取 body

## 设计说明

这个 Creator 背后是一套经过实际学习项目验证的方法论（五步闭环、模块复盘、认知框架沉淀）。Creator 做的事情是把方法论"编译"成一个可执行的 Skill——根据你的具体情况选择合适的结构和粒度。

几个关键设计决策：

- **严禁删除用户文件**：所有"归档"行为都是读取优先级的调整，原始笔记始终保留
- **通知但不打扰**：每次自动保存都会简短通知你，但不需要你做任何操作
- **引导而非强制**：验收可以跳过、复盘可以跳过、模式可以覆盖——Skill 给建议，你做决定
