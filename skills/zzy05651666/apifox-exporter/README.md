# Apifox Exporter

> 将 Apifox 导出的 OpenAPI JSON 文件整理成 AI 友好的标准格式文档

## 🎯 使用场景

**一次性设置，长期便利：**

1. **第一次：** 手动从 Apifox 导出 JSON（1 分钟）
2. **之后：** 每次接口更新，只需说 **"更新接口文档"**（1 秒钟）
3. **自动：** 桌面生成整理好的标准文档

---

## 快速开始

### 📥 第一次：手动导出 JSON

**操作一次（1 分钟）：**
1. 打开 Apifox
2. 选择项目 → 导出 → OpenAPI 3.0 → 保存为 JSON
3. 放到 `~/.openclaw/workspace/script/apifox/source.json`

### 🚀 之后：用口令更新

**每次只需 1 秒：**
```
更新接口文档
```

**或指定项目：**
```
更新接口文档，用商城项目
```

### 📄 查看结果

桌面自动生成：`~/Desktop/Apifox 接口导出.txt`

---

## 💡 为什么这样设计？

**优势：**
- ✅ **安全** - 不需要给 Skill Apifox 账号权限
- ✅ **快速** - 1 秒钟口令 vs 手动整理几小时
- ✅ **灵活** - 支持多项目随时切换
- ✅ **离线** - 不需要网络连接

**对比手动整理：**
| 方式 | 时间 | 质量 |
|------|------|------|
| 手动整理 | 2-3 小时 | 容易出错 |
| 本 Skill | 1 秒钟 | 100% 准确 |

## 多项目管理

**文件结构：**
```
script/apifox/
├── source.json          # 默认项目
├── source-商城.json      # 商城项目
├── source-后台.json      # 后台管理
└── source-小程序.json    # 小程序项目
```

**切换项目口令：**
```
更新接口文档，用商城
更新接口文档，用后台
更新接口文档，用小程序
```

## 输出格式

- 📁 按模块分组
- 📋 带目录索引
- 🔍 响应数据深挖到底
- 📝 完整的请求参数示例

**示例：**
```
════════════════════════════════════════════════════════════════════════════════
【登录 - 认证】 (4 个接口)
════════════════════════════════════════════════════════════════════════════════

1. 使用账号密码登录
   所属模块：登录 - 认证
   接口地址：/system/auth/login  POST

请求参数 application/json：
{
  "username": "admin",
  "password": "123456",
  "rememberMe": false
}

响应数据：
{
  "code": 0,
  "data": {
    "userId": 0,
    "accessToken": "",
    "refreshToken": "",
    "expiresTime": ""
  },
  "msg": ""
}
```

## ⚠️ 安全提示

**敏感数据警告：**
- 导出的文档会原样输出 OpenAPI JSON 中的内容
- 如果 JSON 包含真实密码、token 等敏感信息，会出现在输出文件中
- **建议：** 在 Apifox 导出前删除敏感示例数据

**跨平台支持：**
- Windows: `C:\Users\用户名\.openclaw\workspace\`
- Mac/Linux: `~/.openclaw/workspace/`
