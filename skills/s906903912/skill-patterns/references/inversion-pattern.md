# Pattern 4: Inversion（逆向采访）

## 核心作用

**先采访再执行**，防止 Agent 在需求不明确时瞎猜，强制收集完整上下文。

## 适用场景

- 需求模糊的项目规划
- 复杂系统设计
- 个性化任务（需要用户偏好）
- 多约束条件的任务

## 目录结构

```
skills/project-planner/
├── SKILL.md
└── assets/
    └── plan-template.md      # 最终输出模板（可选）
```

## SKILL.md 模板

```markdown
---
name: project-planner
description: 通过结构化提问采集需求，然后生成项目计划。当用户说"我想构建"、"帮我规划"、"设计一个系统"、"开始新项目"时激活。
metadata:
  pattern: inversion
  interaction: multi-turn
  trigger-phrases: [我想做，帮我规划，设计一个，build a, plan a, design a system]
---

你正在进行结构化需求采访。**严格遵守以下规则**：

## ⛔ 禁止事项
- **禁止**在采访完成前开始设计或编写代码
- **禁止**一次性问所有问题（一次只问 1 个）
- **禁止**跳过任何问题

## ✅ 必须事项
- 每次只问 1 个问题，等待用户回答后再问下一个
- 如用户回答模糊，追问澄清
- 采访完成后，总结需求请用户确认

---

## Phase 1 — 问题发现（逐个提问，等待每个回答）

按顺序问以下问题，**不要跳过任何一个**：

**Q1**: "这个项目为用户解决什么问题？"
→ 等待用户回答

**Q2**: "主要用户是谁？他们的技术水平如何？"
→ 等待用户回答

**Q3**: "预期规模？（日活用户、数据量、请求频率）"
→ 等待用户回答

---

## Phase 2 — 技术约束（Phase 1 全部回答后才开始）

**Q4**: "使用什么部署环境？（本地/云/边缘）"
→ 等待用户回答

**Q5**: "有技术栈偏好吗？（语言/框架/数据库）"
→ 等待用户回答

**Q6**: "不可妥协的要求？（延迟/可用性/合规/预算）"
→ 等待用户回答

---

## Phase 3 — 需求确认（所有问题回答后）

1. 总结收集到的所有需求
2. 问用户："以上需求总结准确吗？有遗漏或需要调整的吗？"
3. 根据用户反馈迭代，直到用户确认

---

## Phase 4 — 生成计划（用户确认需求后）

1. 加载 `assets/plan-template.md` 获取输出格式
2. 用采集的需求填充每个章节
3. 呈现完整计划
4. 问："这个计划是否符合你的预期？需要调整什么？"
5. 根据反馈迭代，直到用户确认
```

## 变体：简化版 Inversion

适用于轻量级任务：

```markdown
## 快速采访（3 个问题）

在开始之前，我需要了解：

1. "你的主要目标是什么？"
2. "有什么技术约束或偏好？"
3. "如何定义成功？（验收标准）"

请依次回答，然后我会给出方案。
```

## 变体：分支式采访

根据用户回答动态调整问题：

```markdown
## Phase 1 — 问题发现

**Q1**: "这个项目的主要问题是什么？"

→ 如用户回答"性能问题"，进入性能优化分支
→ 如用户回答"功能缺失"，进入功能设计分支
→ 如用户回答"成本太高"，进入成本优化分支
```

## 对话状态追踪

对于复杂采访，建议追踪状态：

```markdown
## 当前状态
- Phase: 2/3
- 已回答问题：Q1, Q2, Q3, Q4
- 待回答问题：Q5, Q6
- 需求已确认：否
```

## 优缺点

| 优点 | 缺点 |
|-----|------|
| 避免理解偏差 | 对话轮次多，耗时长 |
| 用户参与感强 | 用户可能不耐烦 |
| 输出更符合预期 | 问题设计需要经验 |

## 与 Generator 组合

```markdown
## Inversion → Generator 流程

Phase 1-3: Inversion 采集需求
  ↓
Phase 4: Generator 填充模板
  ↓
Phase 5: 用户确认 → 如不满足，返回 Phase 1 重新采集
```

## 示例对话

```
用户：我想做一个博客系统

Agent: 好的！我先问几个问题来了解你的需求。

**Q1**: 这个博客系统主要为用户解决什么问题？
      是个人博客、企业内容平台、还是其他类型？

用户：企业内容平台，用于发布技术文章和案例

Agent: 明白了！

**Q2**: 主要用户是谁？他们的技术水平如何？
      是内部员工、客户、还是公开读者？

用户：主要是潜在客户，技术水平中等

Agent: 好的！

**Q3**: 预期规模？比如每天多少访客、多少篇文章？

用户：预计日活 1000 人，初期约 50 篇文章

Agent: ...（继续 Phase 2 技术约束问题）
```
